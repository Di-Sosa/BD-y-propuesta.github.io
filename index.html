Bases de datos

Un base de datos es un conjunto organizado y estructurado de información almacenada en un sistema informático, diseñado para gestionar, recuperar, actualizar y analizar datos de manera eficiente. Los datos se organizan en tablas (en bases de datos relacionales), documentos (en bases NoSQL), grafos u otros modelos, según el tipo de base de datos.

    Características principales:

  • Estructuración: Los datos se almacenan siguiendo un esquema definido (por ejemplo, tablas con filas y columnas en bases relacionales).
  • Gestión centralizada: Se utiliza un Sistema de Gestión de Bases de Datos (DBMS) como MySQL, Oracle o MongoDB para administrar el acceso, la seguridad y las operaciones.
  • Consulta eficiente: Permite buscar y filtrar información rápidamente mediante lenguajes como SQL (para bases relacionales) o métodos específicos en bases NoSQL.
  • ntegridad y seguridad:I Garantiza la precisión de los datos mediante reglas de validación, claves primarias, foráneas y mecanismos de autenticación.
  • Escalabilidad: Puede adaptarse a grandes volúmenes de datos y usuarios simultáneos.

    Tipos de bases de datos:

  • Relacionales (SQL): Utilizan tablas y relaciones entre ellas. Ejemplos: MySQL, PostgreSQL.
  • NoSQL: Diseñadas para manejar grandes volúmenes de datos no estructurados. Ejemplos: MongoDB, Cassandra.
  • Orientadas a objetos: Almacenan objetos complejos (ej: PostgreSQL con extensiones).
  • En memoria:Optimizadas para velocidad (ej: Redis).
  • Distribuidas: Datos replicados en múltiples servidores (ej: Google Spanner)

Relacionales (SQL)

Características:

Estructura en tablas: Datos organizados en tablas con filas (registros) y columnas (campos).

Esquema rígido: Requieren definir la estructura (tablas, tipos de datos, relaciones) antes de almacenar datos.

SQL (Structured Query Language): Lenguaje estándar para consultas.

ACID: Garantizan transacciones seguras (Atomicidad, Consistencia, Aislamiento, Durabilidad).

Relaciones: Usan claves primarias y foráneas para vincular tablas.

Ejemplos de Sistemas:

MySQL, PostgreSQL, Oracle, Microsoft SQL Server.

Ejemplo Práctico:

Imagina una base de datos para una tienda online:

Tabla Usuarios:

id (PK) nombre email
1 Ana ana@example.com
2 Carlos carlos@example.com

Tabla Pedidos:

id (PK) usuario_id (FK) producto cantidad
101 1 Laptop 1
102 2 Teléfono 2

Relación:

usuario_id en Pedidos referencia al id en Usuarios.

Consulta SQL:

SELECT Usuarios.nombre, Pedidos.producto
FROM Usuarios
JOIN Pedidos ON Usuarios.id = Pedidos.usuario_id
WHERE Usuarios.id = 1;

Resultado:

nombre producto
Ana Laptop

Estructura de Tablas

CREATE TABLE Usuarios (
    id INT PRIMARY KEY,
    nombre VARCHAR(50),
    email VARCHAR(100)
);

CREATE TABLE Pedidos (
    id INT PRIMARY KEY,
    usuario_id INT,
    producto VARCHAR(50),
    FOREIGN KEY (usuario_id) REFERENCES Usuarios(id)
);

NoSQL

Características:

Flexibilidad: No requieren un esquema fijo (ideal para datos no estructurados o semi-estructurados).

Escalabilidad horizontal: Distribuyen datos en múltiples servidores.

Modelos variados:

  • Documentos (JSON/BSON): Ej: MongoDB.
  • Clave-Valor: Ej: Redis.
  • Columnas: Ej: Cassandra.
  • Grafos: Ej: Neo4j.

Ejemplos de Sistemas: MongoDB, Cassandra, Redis, Couchbase.

Ejemplo Práctico (MongoDB - Documentos):

En una red social, se almacenan perfiles de usuarios con datos variables:

Colección Usuarios:
{
    "_id": "abc123",
    "nombre": "Ana",
    "email": "ana@example.com",
    "intereses": ["tecnología", "viajes"],
    "amigos": [
        {"id": "def456", "nombre": "Carlos"},
        {"id": "ghi789", "nombre": "Luisa"}
    ],
    "actividad": {
        "ultimo_login": "2023-10-25",
        "posts": 15
    }
}
Consulta en MongoDB (Buscar usuarios interesados en "viajes"):

db.Usuarios.find({ "intereses": "viajes" });


Ventaja: Los datos anidados (como amigos o actividad) se almacenan en un solo documento, evitando joins complejos.
